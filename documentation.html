Overview

The Book Template Generator is a comprehensive web application designed to assist authors, designers, and publishers in creating accurate print-ready templates for various types of books. It dynamically calculates critical dimensions such as spine width, full cover size (including bleed or wrap), safety margins, and hinge widths based on user inputs like page count, paper stock (PPI), trim size, and binding type.

The application supports:

  • Perfect Bind / Softcover
  • Case Bind / Hardcover
  • Coil / Wire-O - Softcover
  • Coil / Wire-O - Hardcover

Beyond cover templates, it provides an interactive interior page setup guide with recommended gutter margins and tools to generate interior document templates. A key feature is the integrated barcode generator, allowing users to create and download ISBN EAN-13+5 or Data Matrix ECC 200 barcodes for their book covers. The goal is to streamline the pre-press process, reduce errors, and empower users to generate professional-quality templates quickly.

Features

  • Dynamic Cover Dimension Calculation: Accurate calculations for:
    • Spine Width
    • Total Cover Width & Height
    • Bleed or Wrap Amounts
    • Safety Margins
    • Hinge Widths (for Case Bind)
    • Board Dimensions (for Case Bind & Hardcover Coil/Wire-O)
  • Binding Type Support: Handles Perfect Bind, Case Bind, Coil/Wire-O Softcover, and Coil/Wire-O Hardcover.
  • Interactive Previews:
    • Visual preview of the generated cover template with technical guides.
    • Interactive interior page spread preview (Verso/Recto) with dynamic gutter visualization.
  • Downloadable Assets:
    • PDF cover templates with guides.
    • Adobe Photoshop (.jsx) script for automated cover template setup.
    • Adobe InDesign (.jsx) script for automated cover template setup.
    • Microsoft Word (.docx) interior page template with appropriate page size, bleed, and margins.
    • Adobe InDesign (.jsx) script to create an IDML interior page template.
    • Convenient ZIP archives for "Cover Files," "Interior Files," or "All Files."
  • Custom Barcode Generator:
    • Supports ISBN EAN-13 (+5 digit price add-on) and Data Matrix ECC 200.
    • Real-time preview of generated barcodes.
    • Download options for barcodes in JPEG and PDF formats.
    • Automatic ISBN-10 to ISBN-13 conversion and check digit validation.
  • User Experience:
    • Responsive design for various screen sizes.
    • Theme switching (Light, Dark, System preference).
    • Collapsible UI sections for improved layout organization.
    • "Copy to Clipboard" functionality for the book summary.
    • Clear input validation and user-friendly error messages.
  • Technical Guides:
    • Togglable technical guides on cover previews.
    • Detailed interior layout and typography guidelines.
    • Explanations for gutter margins and file specifications.

Tech Stack

  • Frontend:
    • React 19 (with TypeScript)
    • HTML5
    • Tailwind CSS (via CDN for styling)
  • Core Libraries:
    • jspdf: For generating PDF documents (cover templates, barcode PDFs).
    • jszip: For creating ZIP archives of downloadable files.
    • bwip-js: For generating barcode images (ISBN, Data Matrix).
    • docx: For generating Microsoft Word (.docx) interior templates.
    • @google/genai: For integrating with Google's Gemini API (Branding Hub feature - currently commented out).
    • html2canvas: (Included in importmap, potentially for image rendering or used by a dependency).

Prerequisites

  • A modern web browser that supports ES modules (e.g., Chrome, Firefox, Safari, Edge).
  • For using the AI-powered "Branding Hub" feature (currently commented out):
    • A valid API Key for the Google Gemini API. This key needs to be available as process.env.API_KEY in the execution environment.

Installation & Setup

This project is designed to run directly in the browser without a complex build step for its core functionality.

  1. Clone the repository: 
    git clone https://your-repository-url/book-template-generator.git
  2. Navigate into the project directory: 
    cd book-template-generator
  3. Environment Variables (Optional - for AI features):

    If you plan to enable and use the "Branding Hub" feature which uses the Gemini API:

    • The application expects the Gemini API key to be available via process.env.API_KEY. How this is set depends on your hosting or local serving environment if you intend to use AI features. For simple static hosting, this is non-trivial without a backend or build process. For deployments, configure this in your hosting provider's settings.
  4. Run the application:
    • Open the index.html file directly in your web browser.
    • OR, for a better experience and to avoid potential issues with file paths or module loading, serve the project root directory using a simple local HTTP server. For example, using Node.js serve: 
      npx serve .
      Then open http://localhost:3000 (or the port indicated by serve) in your browser.

Usage / Running the App

  1. Ensure you have followed the Installation & Setup steps.
  2. Open index.html in your browser or navigate to the local server address if you started one.
  3. Use the form in the "Book Specifications & Summary" section to input your book's details:
    • Book Title (Optional)
    • Trim Width & Height
    • Binding Type
    • Page Count and Paper Stock (required for binding types that calculate spine width)
  4. Click "Calculate & View Options".
  5. The "Previews & Setup Guides" section will update with:
    • A visual preview of your cover template.
    • An interactive interior page setup guide.
    • Download buttons for various template files (PDF, JSX scripts, DOCX, ZIP).
  6. Optionally, use the "Generate Barcode Image" section to create and download ISBN or Data Matrix barcodes.
  7. Use the theme switcher in the header to toggle between Light, Dark, and System themes.

API Endpoints

This is a frontend-only application. It interacts with the Google Gemini API (if the Branding Hub feature is enabled) which is an external service. No backend API is hosted by this project itself.

Deployment

This application is a static frontend website and can be deployed to any static site hosting provider.

Steps:

  1. Ensure all paths in index.html (especially for src/index.tsx) are correct for your deployment environment (usually root-relative paths like /src/index.tsx are fine).
  2. Upload the entire project folder (or just the src directory and index.html, metadata.json if your host serves index.html from root) to your chosen static hosting service. Popular choices include:
    • Netlify
    • Vercel
    • GitHub Pages
    • AWS S3 with CloudFront
    • Firebase Hosting
  3. Environment Variables for AI Features: If you are using the Gemini API (Branding Hub), you must configure the API_KEY as an environment variable in your hosting provider's settings. Refer to your hosting provider's documentation on how to set environment variables for frontend builds or functions if you decide to proxy API calls.

No special build commands are currently required for deployment as is.
